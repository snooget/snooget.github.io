Intro to Semantic html

When building webpages we use a combination of semantic and non semamtic html. The words semantic stand for
meaningful.

With semantic elements there is more information explaining the code in between the elements. Semantic html
uses elements based on their meaning. For example, div and span are used to divide lines of code but do not give
us any other information as to what the code inside of the tags mean.

For example, we can use a "header" element to suggest that the information inside the code is part of the website head.


Why use semantic html?

  • Accessibility: Semantic html makes webpages more accessible for mobile devices and disabled people.
  • SEO: Using semantic html makes website crawlers understand the format of your site, resulting in higher SEO ratings.
  • Makes Source Code Easy to Understand: Using semantic html makes it easy for you and other developers to read
    the source code.

    • Think of semantic html as labels in a store or shopping center. It just makes websites more clear.
