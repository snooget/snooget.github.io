Introduction: Git & GitHub

I will practice using basic Git commands in the terminal to add version control to my projects. This will allow me to
switch between different versions of my project. Git is super useful for creating new features for a project while keeping
an extra copy of the old version.

In a few more pages I will go over how to integrate Git with GitHub, which is one of the most used web applications
by developers across the world to collaborate on projects and share code!

By the end of this project I will be able to:

  • Create a Git project & setup a remote copy on GitHub.
  • Label code changes and move between different versions of the project.
  • Write text on the GitHub interface using markdowns to describe project & code changes.
  • Create branches in your Git project so you can collaborate with others.
  • Use Git commands to make sure your local copy of code matches the remote copy on Github.
  • Use GitHub pull requests to discuss code with others.
  • Be familiar with Git rebase, Git repository settings, and how to keep a remote repository organized.
  • How to create a GitHub profile and explore code written by others.
  • Be familiar with GitHub features that allow you to manage tasks within GitHub itself and automate actions.
