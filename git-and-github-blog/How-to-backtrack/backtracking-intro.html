Backtracking Intro Notes

When working on a Git project, sometimes we make changes that we want to get rid of. Git offers, a few
features that allow us to undo mistakes during project creation.

Head commit

In Git the commit you are currently on is called the HEAD commit. In many cases the most recent commit
is the HEAD commit. To see a HEAD commit enter:

git show HEAD

git checkout

This is useful for when you make a change and decide you would like to undo it. You may have forgotten a
line of code and to return/undo it to the original wording you would use the following git command:

git checkout HEAD filename

This command will restore the file in your working directory, to look exactly as it did when you made a commit.

git add

In git, it is common to change many files, add thoes files to the staging area, and commit them to a repository in
a single commit.

For example, lets say you would like to replace the word LARRY with LAERTES. If the name was contained in two
files you can edit them in editor, then add them to the staging area with:

git add filename_1 filename_2

git reset I

The files we previously added to the same commit were a success, and now belong to the same commit. What if,
before you commit, you accidently delete an important scene from one of the txt files? Unthinkingly adding, the
scene-2.txt to staging area. The file change is unrelated to the Larry/Laertes swap, and not included in the commit.

A file can be unstaged from the staging area by using:

git reset HEAD filename

The command above resets the file in the staging area to be the same as the head commit. It does not discard changes
from the working directory, it just removes them from the staging area.

git reset II

Creating a project is like hiking in a forest. Sometimes you take a wrong turn and find yourself lost. Just like
retracting your steps on a hike, git enables you to rewind to that part before you made the wrong turn. Look:

git reset commit_SHA

This command works by using the first 7 characters of the SHA of a previous commit. For example, if the SHA of a
previous commit is 5d692065cf51a2f50ea8e7b19b5a7ae512f633ba ,use:

git reset 5d69206

HEAD is now set to the previous commit.
