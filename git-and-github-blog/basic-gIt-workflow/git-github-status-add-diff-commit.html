Git & GitHub Workflow Notes Add, Diff, and Commit


git status

As you work, you will change the contents of the wokring direcotry. Use "git status" to view the changes.

git status

git add

In order for git to start tracking, files must be added to the staging area. We can use "git add" to add a file
to the staging area.

git add filename

The word filename above stands for the name of your files in that space.

git diff

With git add we added files to the staging area. Since added files are now tracked by Git,
we can check the differences between the working directory and staging area with the command below:

git diff filename

git commit

git commit is the last step in our workflow. A git commit permanetly stores changes from staging
area to the repository.

git commit is followed with an -m option.
Here's an example:

git commit -m "complete first line of dialouge"

Here are some standard conventions to follow when using git commit commands.

  • Must be in quotation marks.
  • Written in present tense.
  • Should be breif, 50 characters or less when using -m.

git log

Finally we have git log which checks all previous versions of a project. Commits are stored
chronologically in repository."git log" ahows them.
