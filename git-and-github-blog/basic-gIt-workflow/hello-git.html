Hello Git | Git init | Git Workflow

Git is a software that allows you to keep track of changes to a project over time. Git works by recording changes
then storing them to become accessable for future reference or use.

Git init

The word init means initialize. The command sets up all the tools Git needs to begin tracking changes made to a
project.

Git Workflow

Once we "git init" we have started our Git project! A git project can be divided into three parts:

  1. A working directory: Where you'll be doing all the work. Creating, editing, deleting, and organizing files.
  2. A staging area: Where you'll list changes you make to the working directory.
  3. A repository: Where Git permanantly stores thoes changes as different versions of the project.

The git workflow consist of editing files in the working directory, adding files to the staging area, and saving
changes in the repository. In Git we save changes with a commit, which will be covered below.
