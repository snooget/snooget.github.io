git team generalizations

Here is a quick review of the git team module:

  • A remote is a Git repository that lives outside your Git project folder. Remotes can live on the web, on a
    shared network or even in a seperate folder on your local computer.

  • The Git collaborative workflow are steps that enable smooth project development when multiple collaborators
    are wokring on the same project.

    • We also learned the following commands:

  • git clone: Creates a local copy of the remote.

  • git remote -v: List's a Git projects remotes.

  • git fetch: Fetches work from the remote into the local copy.

  • git merge origin/master: Merges origin/master into your local branch.

  • git push origin branch_name: Pushes a local branch to the origin remote.
