Handy Git Operations

Intro

Git provides us with a vast number of different commands listed in the documentation. Here are a few.

Git Stash

Let's say you are working on experimental code in a fresh new branch. After some time you realize that you forgot
to add something to the previous commit.

In order to go on to a different branch one must always be at a clean commit. In this example you haven't commited
your experimental code as it is not ready. However, you don't want to loose all the code you've been working on.

A good way to handle this situation is to use git stash as it allows you to get back to a clean commit
point wihtout loosing your experimental code work.

Once you fix up the code on the previous commit, you can use git stash pop to continue off on the work
that was previously stashed.

git log

We are quite familiar with git log. However, there are a few more ways to use the log command than I know.

git log --oneline will show the list of commits in one line format.

git log -s "keyword" displays a list of commits that contain the keyword in the message.

git log --oneline --graph -- graph shows a visual representation of how the branches and commits
were created in order to help you make sense of your repository. Using --oneline makes the --graph command shorter.

git commit ammend

Git's --amend flag is really useful when updating a commit. It allows for correction and editing of commits
without having to create entire new ones.

For example, you have just finished committing a lenghty feature, and find out after that there is a syntax error
causing a bug. To avoid creating a new commit, you can create the changes, stage them with git add, and then
type the command git commit --amend.

Remember that although it seems git commit --amend is updating the last commit, what is really
happening is an entire replacement. So when using the above command, your terminal will make you update your commit
message.

However, if you want to keep the same commit message then make sure to write:

git commit --amend --no-edit

git alias commands

When grouping commands together you can end upo writing long lines of code such as:

$ git log --pretty=format: "%h %s" --graph

Fortunately, Git offers a helpful feature that can make your Git experience simpler, easier, and more familiar.
aliases!

If you have a set of commands you use regularly and would want to save some time from fully typing them, you can
easily set up an alias for each command using Git config.

Below are a few examples:

$ git config --global alias.co "checkout"
$ git config --global alias.br "branch"
$ git config --global alias.glop "log --pretty=format: "%h %s" --graph"
