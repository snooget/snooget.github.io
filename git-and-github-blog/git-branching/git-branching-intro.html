git Branching Intro

Most of the work I have done for git has been on a single branch.
I will learn how to switch from main to other branches with git commands.

git allows us to create seperate branches in order to experiment with code
and create different versions of a project.

To check the current branch type:

$ git branch

git Branching II

Many projects will have a master commonly called main branch. To create a
new branch, use:

$ git branch new_branch

The words "new_branch" will be something that describes the purpose of the
branch. Remember that branch names can't contain whitespace.

$ git Checkout

$ git checkout branch_name

The command above allows you to switch branches and make new edits under the
selected branch.

When switching branches you are able to do all commands that you where able to
previously do in the main branch.

$ git merge

Once you are satisfied with the edits on the other branch you can then merge the
content from the named branch to the main branch.

This is accomplished by typing:

$ git merge branch_name

merge conflict I

Merge conflicts occur for a few reasons, here is one: Imagine you made commits on
the main branch before merging the secondary branch. Additionally, what if you made a
commit on main that alters the same exact text you worked on in the secondary branch. This
is how merge conflicts emerge. Here's how to solve them:

A merge error will appear when you do as described above. To fix it you will refresh the
text editor and see two versions of the edit. You can select which one you prefer then create
a new commit, and viola! Problem solved!

delete branch

git branches are a means to an end and are meant to be merged into the main branch.
The command:

$ git branch -d branch_name

is used to delete a branch once it has been merged.
