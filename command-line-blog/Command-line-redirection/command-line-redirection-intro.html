Command Line Redirection Intro

Introduction

Up until now I have been running commands in the comamnd line and have received a stream of
output in the terminal. In these next lessons I will focus on input/output (I/O) redirection.

Through redirection I can direct the input and output of a command to and from other
files and programs. Which will allow me to chain commands together into a pipeline.


$ echo "Hello"
Hello

The (echo) command accepts the string "Hello" as standard input, and echos the string
"Hello" back to the terminal as standard output.

Let's learn more about standard input, standard output, and standard error:

  1. standard input, abbreviated as (stdin), is information inputted into the
    terminal through the keyboard or input device.

  2. standard output, abbreviated as (stdout), is the information outputted after
    a process is run

  3. standard error, abbreviated as (stderr), is an error message outputted by a
    failed process.

    4. Redirection reroutes standard input, standard output, and standard error to or from
    a different location.
