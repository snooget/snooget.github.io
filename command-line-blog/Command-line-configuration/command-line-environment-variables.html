Environment Variables & PS1

Environment variables can be used across commands and programs that hold information about the environment.
What happens when you store the following command below in $ ~/.bash_profile?

$ export USER="Jane Doe"

  1. The line $ USER="Jane Doe" sets the environment variable to a user named Jane Doe.
  2. The line export makes the variable to be avaliable to all child sessions initiated
    from the session you are in. This makes the variable persistant.
  3. At the command line, the command $echo $USER returns the value of that variable. Note
    that $ is always used when returning a variables value. Here, the command echo $USER returns the name set for
    the variable.

PS1 Variable

The PS1 variable defines the makeup and style of of the command prompt.
What happens when this is stored in the $ ~/.bash_profile?

$ export PS1=">>"

  1. $ export PS1=">>" sets the command prompt variable and exports the variable. Here we change
    the default command prompt from $ to >>.
  2. After using the source command, the command line displays the new command prompt.

The HOME Environment Variable

What happens when I type the command below?

$ echo $HOME

The HOME environment variable displays the path of the home directory ~. The HOME variable can be specified & changed.

By typing $ echo $HOME, the terminal displays the path /home/ccuser as output.
